Remapping Python Refactor (HILDA+/MIRCA/AgGRID)
================================================

Overview
--------
This package refactors the MATLAB remapping pipeline into Python with
parity-focused behavior. It ingests land-use (HILDA+ or LUH2), crop
fractions (MIRCA), and nitrogen fertilizer (AgGRID) data, interpolates
and regrids to the target gridlist, then writes LU, cropfracs, and nfert
outputs in LPJ-GUESS style tables.

Entry Points
------------
- `remap.py`: main pipeline driver (`run_remap`)
- `remap_options.py`: config dataclass (`RemapConfig`)
- `tests_py/run_parity.py`: parity checks against MATLAB outputs

Setup
-----
- Install the package: `pip install -e .` from the `landsymm_py/` root.
- Ensure Python has `numpy`, `pandas`, `scipy`, `netCDF4`, and `h5py`
  (installed automatically via `pip install -e .`).

Quick Run
---------
Use the same options as MATLAB `remap_options.m`:

```
python - <<'PY'
from landsymm.remapping.remap_options import RemapConfig
from landsymm.remapping.remap import run_remap

from landsymm.config import get_geodata_dir, get_remap_output_dir

geodata = get_geodata_dir()
gridlists = geodata / "gridlists"
soil = geodata / "soil"

cfg = RemapConfig(
    geodata_dir=str(geodata),
    year_list_out=list(range(1901, 2021)),
    lu_source="HILDA+",
    out_dir_top=str(get_remap_output_dir()),
    fill_unveg=1e-6,
    interp_test_y1=1995,
    interp_test_yN=2005,
    inpaint_method=4,
    force_all_rainfed=False,
    plum_setaside_frac=0.088,
    remap_ver="10_old_62892_gL",
    this_ver="WithFruitVeg_sepOil_combSugar",
    file_gridlist_out=str(gridlists / "gridlist_62892.runAEclimOK.txt"),
    file_gridlist_climate=str(
        gridlists / "remaps_v10_g2p_isimipclimMask"
        / "gridlist.remapv10_g2p_isimipclimMask.txt"
    ),
    files_soil=(
        str(soil / "HWSD_soil_data_on_cropland_v2.2.remapv10_old_62892_gL.txt"),
        str(soil / "HWSD_soil_data_on_cropland_v2.3.remapv10_old_62892_gL.txt"),
        str(soil / "soilmap_center_interpolated.dat"),
    ),
    out_prec=6,
    out_width=1,
    delimiter=" ",
    overwrite=True,
    fancy=False,
    do_gzip=False,
    log_file=None,  # defaults to out_dir/remap.log
)

run_remap(cfg)
PY
```

Reusable Run Script
-------------------
A reusable script is included at `landsymm.remapping/run_remap.py`.
Edit the configuration values in that file as needed, then run:

```
cd landsymm_py
python -m landsymm.remapping.run_remap
```

- Static data:

- Gridlists:
  `data/geodata_py/gridlists/*`

MIRCA.txt and the .maps.mat Cache
----------------------------------
The code references `MIRCA.txt` as the canonical input path, but `read_table_then2map()`
in `lpjg_io.py` transparently uses a `.maps.mat` cache when available:

1. On first call, it checks for `MIRCA.txt.maps.mat`. If present and valid, it loads
   the pre-computed map data (fast path, ~instant).
2. If the cache is missing or corrupt, it falls back to reading `MIRCA.txt` directly,
   converts to map form, and saves a new `.maps.mat` cache for next time.

`MIRCA.txt` was historically generated by MATLAB preprocessing scripts from the raw
MIRCA2000 rasters. A Python reconstruction script is included at
`landsymm/remapping/rebuild_mirca_txt.py`. It reads the 52 original `.asc.gz` rasters
(26 irrigated + 26 rainfed) from `MIRCA/harvested_area_grids_26crops_30mn/originals/`
and produces a byte-exact replica:

```
python -m landsymm.remapping.rebuild_mirca_txt          # rebuild
python -m landsymm.remapping.rebuild_mirca_txt --verify  # rebuild + validate vs cache
```

Outputs
-------
Written under `out_dir_top/remaps_v{remap_ver}`:
- `LU.remapv{remap_ver}.txt`
- `cropfracs.remapv{remap_ver}.txt`
- `nfert.remapv{remap_ver}.txt`
- `gridlist.remapv{remap_ver}.txt`

Logging
-------
By default, a log is written to `out_dir_top/remaps_v{remap_ver}/remap.log`.
To override, set `log_file` in `RemapConfig`.

Configuration Reference (RemapConfig)
-------------------------------------
- `geodata_dir`: root geodata path (preferred source for all inputs)
- `year_list_out`: list of years to output (e.g., `range(1901, 2021)`)
- `lu_source`: `"HILDA+"` or `"LUH2"`
- `out_dir_top`: output root directory
- `fill_unveg`: fraction to transfer from BARREN to NATURAL in unvegetated cells
- `max_nan_frac`: maximum fraction of gridlist cells allowed to have no HILDA+ data before erroring (default `0.01`). Cells below the threshold are set to BARREN=1.0 and handled by `fill_unveg`.
- `interp_test_y1`, `interp_test_yN`: years for interpolation sanity checks
- `inpaint_method`: method for inpainting (`4` = spring analogy)
- `force_all_rainfed`: if true, merge irrigated into rainfed for crops
- `plum_setaside_frac`: PLUM set‑aside fraction for crop fractions
- `remap_ver`: output version tag (e.g., `10_old_62892_gL`)
- `this_ver`: crop mapping version string
- `file_gridlist_out`: output gridlist file
- `file_gridlist_climate`: climate gridlist (optional diagnostics)
- `files_soil`: soil input file list
- `out_prec`, `out_width`, `delimiter`: output formatting
- `overwrite`, `fancy`, `do_gzip`: output behavior
- `log_file`: optional log file path (default: `out_dir/remap.log`)

Known Issues / Diagnostics
--------------------------
- `LU` parity: tiny 1e‑6 scale differences remain (within output precision).
- `cropfracs` and `nfert`: exact parity against MATLAB outputs.
- Log file includes full progress trace and is the first place to look if a run stops.

Pipeline Steps (high-level)
---------------------------
1. Load gridlists and config options.
2. Interpolate soil properties to output grid.
3. Import LU (HILDA+ or LUH2), map classes, add water fraction, handle unveg.
4. Import MIRCA crop fractions, map crops, apply setaside, build ExtraCrop.
5. Import AgGRID Nfert, map crops, enforce rainfed=irrigated.
6. Interpolate cropfracs/nfert to output grid, normalize, and save.

Parity Checks
-------------
Remap-only parity (with detailed diff stats):

```
python -m tests_py.run_parity \
  --remap-matlab "data/output_hildaplus_remap_10b/remaps_v10_old_62892_gL" \
  --remap-python "data/output_hildaplus_remap_10b/remaps_v10_old_62892_gL" \
  --skip-plum \
  --remap-details
```

Notes on Warnings
-----------------
The pipeline suppresses non-actionable warnings related to:
- cached `.mat` tables missing headers (falls back to text)
- conflicting cache save flags (mat save disabled)
- division by zero in cropfrac normalization (filled via interpolation)
- Qt/Wayland warnings (headless use)

Current Parity Status (remap)
-----------------------------
- `cropfracs` and `nfert`: exact match
- `LU`: tiny differences at 1e-6 scale (within output precision)
